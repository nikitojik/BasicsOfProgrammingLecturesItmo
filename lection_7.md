## анонимные namespace - namespace без названий
Анонимные (или безымянные) пространства имен в C++ используются для объявления идентификаторов, которые локальны для данного файла, предотвращая их столкновение с другими объектами с такими же именами в других файлах. Это достигается путем предоставления им внутренней компоновки, что является лучшей альтернативой использованию устаревшего ключевого слова static для тех же целей. 

### Для чего используются
#### Изоляция идентификаторов: 
Позволяют объявлять переменные, функции и другие сущности, которые доступны только в пределах одного .cpp файла.

#### Внутренняя компоновка: 
Идентификаторы в анонимных пространствах имен имеют внутреннюю компоновку, аналогичную переменным, объявленным с помощью static в C.

#### Избежание конфликтов имен: 
Устраняют необходимость создавать именованное пространство имен просто для локализации одной или нескольких сущностей. 

### Преимущества перед static
Устранение двусмысленности: Ключевое слово static в C++ может иметь разные значения в разных контекстах (например, статическая переменная или статическая функция), что может вызывать путаницу.

Современный стандарт: Использование анонимных пространств имен является более современным и предпочтительным подходом согласно современным стандартам C++.
