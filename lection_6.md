# Reference, initialization, function overloading namespaces

## Reference (ссылки)
- "Псевдоним" для уже существующего объекта
- ссылка обязана быть инициализирована 
- не занимает доп. память
- нельзя сделать указатель на ссылку
- продлевает жизнь временным переменным
- могут быть const(можно сделать на неконстантную переменную)

```cpp
int main(){
    int i = 10;
    int& j = i; // int& ccылка на int
    j = 20;
    std::cout << i << std::endl;
    std :: cout << &i << ' ' << &j <<  std::endl;
    return 0;
}
```


## Передача аргументов в функцию
- по значению (дорого) 
- через указатель (может быть нулевым)
- по const ссылке
- по ссылке

    
## Ссылка на ссылку 
```cpp
int&& rv = 1; - ссылка 
```
Это означает, что первая ссылка (&) является псевдонимом исходной переменной, а вторая ссылка (&&) является псевдонимом для первой ссылки. Такая конструкция используется для работы с универсальными ссылками или ссылками на временные объекты (rvalue), что позволяет функциям обрабатывать как обычные переменные, так и временные значения более гибко.
