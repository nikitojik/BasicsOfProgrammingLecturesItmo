### Абстрактный класс
Класс, в котором есть хотя бы один нереализованный виртуальный метод.

```cpp
class Loger {
public:
 virtual void Log(const char* message) = 0;
};
```

Отсутствие реализации возможно с помощью записи для определения метода:
```cpp
<signature> = 0;
```

### Виртуальный деструктор

```cpp
Base* base = new Inherited();
delete* base; // Inherited's destructor not called
```

Чтобы исправить эту проблему нужно в базовом классе сделать минимальный виртуальный деструктор:
```cpp
class Base {
public:
 virtual ~Base() = default;
};
```

### Стоимость виртуальных функций
- Лишнее обращение к таблице вместо явного адреса;
- Не возможно сделать inline optimization;
- Для коллекций объектов - они всегда в куче;
- Порядок объектов также может влиять на скорость.
