## Компиляция
### компилятор
- поступает исходный код на с++ после препроцессора
- с помощью компилятора можно получить наш код в двоичном виде или в assembler-е.
- на этапе компиляции не важно, существует ли реализация ф-ции, главное, чтобы было объявление.
- анонимный namespace на этапе линковки запрещает передачу в другие объектные файлы
- в заголовочный файл кладем только то, что нужно во всех файлах.


storage duration - время жизни 
- automatic
- static - длится от запуска программы до ее завершения
- thread
- dynamic - управляем мы сами (new/delete)

### storage class specifier
**static** - static duration and internal linkage - доступ к этой переменной только из
текущей единицы трансляции. По умолчанию глобальная переменная internal linkage

**extern** - static duration and external linkage - является глобальной для всех единиц трансляций. Если менять такую переменную 
в одномм файле, она изменится везде

###linkage
- internal
- external
- no linkage


ключевое слово static, теперь переменная живет НЕ на Stack
переменная со словом static лежит в отдельном сегменте памяти 

- ошибки на стадии компиляции возникают из-за синтаксической ошибки
- ошибки на стадии линковки - не смогли связать вызовы функции с их реализациями
